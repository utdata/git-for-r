4.1 Ignoring files

In our last chapter, we ran the git status command to tell us what Git sees within our project.

Let‚Äôs take another look at those files now by re-running:

Under untracked files you‚Äôll see a list like:

        _quarto.yml
        my-first-git-project.Rproj
        my-first-git-project.html
        my-first-git-project.qmd

Notice that .html file in the list. This .html file is the rendered output from Quarto, which changes every time you click ‚ÄúRender.‚Äù We generally do not save these kinds of rendered files in our repositories since it‚Äôs like saving every draft printout of a paper ‚Äî not helpful.

4.2 üôà Introducing .gitignore

In a Git project, there are several types of files and directories that you typically want to keep out of our repositories to keep things clean, secure, and easy to manage. What you ignore will depend on the type of project and the coding languages you‚Äôre using, but all Github projects use a file called .gitignore to let Git know what to ignore.

In general, the following kinds of files shouldn‚Äôt be tracked in Github:

  1. Sensitive files: Files containing passwords or private information
  2. Big files: Git isn‚Äôt built to keep track of large files.

For R and Quarto projects, this will include the following common files types:

  1. Session and history files: .Rhistory, .RData, .Rproj.user/
  2. Knitting files: .utf8.md, .knit.md
  3. Output/render folders: *_files/ ,_site/, _book/
  4. Editor and IDE Config Files: .Rproj.user/

Don‚Äôt worry if you don‚Äôt fully understand what all of these files are yet. The important thing here is to know is that there are things we should and should not upload to Git, and the way we keep the things we don‚Äôt want in git out is by using a .gitignore file.

4.2.1 Exercise 2: Make a .gitignore

Let‚Äôs make a .gitignore now using the command line. Type the following code into the Terminal to make your new file using the touch command:

Then open .gitignore in the Files pane in RStudio. If you don‚Äôt see the .gitignore in your file pane, you may need to refresh by clicking the refresh icon at the top right of the file pane.

Once you have your blank .gitignore file open, add the following lines:

.Rproj.user/

Let‚Äôs break down what each of these mean:

  1. *.html : Ignore all .html files (rendered outputs)
  2. _site/: Ignore the entire contents of the _site folder
  3. .Rproj.user/: Ignore the hidden RStudio files

Save the file, then check your Git status again:

Now the .html file should be gone from the list of red files Git ‚Äúsees‚Äù because we‚Äôve told it to ignore these types of files. ‚ú®

Before moving forward, let‚Äôs add a few more things to the .gitignore to cover all our bases.

Copy and paste the following code into your .gitignore file.

.Rhistory
.RData
.Rproj.user/

# Quarto
*.html
*_files/
*.utf8.md
*.knit.md
_site/
_book/

# System
.DS_Store

Save and close the file.

4.3 ‚úÖ Add the Files You Do Want (Staging)

Now that we‚Äôve told Git what we don‚Äôt want to keep track of, we need to tell Git to track the rest of our files. We‚Äôll do that by using the add command

The . after the add means ‚Äúadd everything in this folder‚Äù (ignoring anything listed in .gitignore.)

The output should look something like this now:

Notice that the heading has changed to ‚ÄúChanges to be committed‚Äù and files are now shown in green. This is called Staging.
