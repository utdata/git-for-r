11.2 Introducing .gitignore

In a Git project, there are several types of files and directories that you typically want to keep out of our repositories to keep things clean, secure, and easy to manage.

What you ignore will depend on the type of project and the coding languages you‚Äôre using, but all Github projects use a file called .gitignore to let Git know what it shouldn‚Äôt be tracking in the project.

In general, the following kinds of files shouldn‚Äôt be tracked in Github:

  1. Sensitive files: Files containing passwords or private information
  2. Big files: Git isn‚Äôt built to keep track of large files.

For R and Quarto projects, this will include the following common files types:

  1. Session and history files: .Rhistory, .RData, .Rproj.user/
  2. Knitting files: .utf8.md, .knit.md
  3. Output/render folders: *_files/ ,_site/, _book/
  4. Editor and IDE Config Files: .Rproj.user/

Don‚Äôt worry if you don‚Äôt fully understand what all of these files are yet. The important thing here is to know is that there are things we should and should not upload to Git, and the way we keep the things we don‚Äôt want tracked out of Github is by using a .gitignore file.

We‚Äôll set one up for our new project in the next step.

11.3 üìù Exercise: Make a .gitignore

Let‚Äôs make a .gitignore for your new Git project from the command line using the touch command.

Once you have your blank .gitignore file open, add the following lines and save the file:

Let‚Äôs break down what each of these mean:

  1. *.html : Ignore all .html files (rendered outputs)
  2. _site/: Ignore the entire contents of the _site folder

.Rproj.user/

*.utf8.md
*.knit.md

Save and close the file.

Now that we‚Äôve told Git what we don‚Äôt want to keep track of, we need to tell Git to track the rest of our files. We‚Äôll do that by using the add command.
