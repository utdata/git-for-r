11  Installation and setup

üéØ Goal: By the end of this chapter, you‚Äôll have Git and GitHub set up and be ready to create a version-controlled project in RStudio.

11.1 Install the Tools

Before we can get started, we‚Äôll need to make sure that our computers (often referred to as our local machines) are have the tools we need. Let‚Äôs do that by completing the following steps:

  1. Install Git

    1. Go to: https://git-scm.com/downloads
    2. Download and install Git
    3. Default install settings are fine

  2. Sign up for a Github account:

    1. Go to: https://github.com
    2. Click Sign up
    3. Fill in your email and password
    4. Choose a username. Keep this professional enough that you wouldn‚Äôt mind a professor, colleague or future employer seeing it.
    Caution

    Remember, the username you set here will be visible (read: public) in any future projects you push to GitHub. If you choose to use a nickname or alias instead of your real name, keep it professional enough that you wouldn‚Äôt mind future colleagues or a potential employer seeing it if needed to share your work.

11.2 Configure Git

After installing Git, you‚Äôll need to go through a one-time process to configure it with your username and email on your computer. We‚Äôll use the terminal to do this.

Step 1: Start a new terminal session in RStudio

  1. Launch R Studio
  2. Click on the terminal tab to start a new session (if there isn‚Äôt one already).

Tip: The terminal tab is located next to the Console tab. If can‚Äôt find the terminal tab, go to Tools > Terminal > Move Focus to Terminal .

Step 2: Set your Git username

  1. Run the following command from your terminal, replacing ‚ÄúYour Name‚Äù with your name (including the quotation marks):

    git config --global user.name "Your Name"

  2. Check to see if you successfully set your name by running the command:

    git config user.name

You should see a terminal response with the username you entered.

If you didn‚Äôt get the response expected, then try again. Remember, be careful about punctuation.

Step 3: Setting your git email

Now we‚Äôll set your user.email.

The email you enter needs to match the one you used to setup your Github account. If it doesn‚Äôt, you‚Äôll run into permissions issues later.

  1. In your terminal, type and run the following command, but replace the email with your email (including the quotation marks):

    git config --global user.email "email@example.com"
Important

Your git email should be the same as the one you use on GitHub so your commits will be linked correctly. If it doesn‚Äôt match, you may run into permissions issues later on.

Step 4: Set your default branch

Next, we‚Äôll set git to name your default branch ‚Äúmain‚Äù instead of ‚Äúmaster‚Äù as this is a new standard.

  1. In your terminal, run this:

You won‚Äôt get a response unless there was a problem.

Step 5: Ensure a clean git history

Lastly, we‚Äôll set a preferred Git configuration for our UT-Data projects. While the exact behavior is a bit complex, this setting helps keep our commit history clean and is recommended in this detailed explanation.

  1. In your terminal, run this:

11.3 Setup your SSH Keys (Authenticating)

Authentication is just a fancy term to describe proving that you are who you say you are. In other words, it‚Äôs how you ‚Äúsign in‚Äù to your Github using the command line. We need to do this so you can connect your project to your Github account and make changes.

Without authentication, GitHub won‚Äôt let you push changes, clone private repositories, or do other actions that affect your account. It‚Äôs how GitHub keeps your code safe and makes sure only you can access your stuff.

There are a couple of ways to authenticate with Github from your computer. The book Happy Git with R recommends using personal access tokens (basically a long password) and that is fine. We prefer the one-and-done nature of SSH keys, so that is the method we will use in this guide.

Instead of using passwords, SSH keys are basically a lock and key system that have to match in order for your computer to access your Github accounts.

You can think of SSH keys like this:

  • üîí Public key = A ‚Äúlock‚Äù you give to GitHub
  • üîë Private key = Your secret ‚Äúkey‚Äù that opens that lock

Together, they let Github know that it‚Äôs really you (or rather your local machine) who is connecting.

We‚Äôll walk through getting SSH keys setup on your machine in the next steps!

Terminology note

When people (including the authors of this guide) use the terms ‚Äúlocal machine,‚Äù ‚Äúlocal computer,‚Äù or just ‚Äúyour machine,‚Äù they‚Äôre all talking about your personal computer ‚Äî the one you‚Äôre using right now.

It‚Äôs called ‚Äúlocal‚Äù to distinguish it from a remote machine (like a server or computer somewhere else, often in the cloud) that you connect to over the internet.

11.3.1 Enable version control in RStudio

First, we‚Äôll enable version control in RStudio by doing the following:

  1. Launch RStudio.
  2. Go to the menu Tools > Global Options, then click on the Git/SVN tab.
  3. Check the Enable version control interface for RStudio button.
  4. There should already be a path included in the git executable field.
  5. Make sure the path to the git executable is correct.
    • If it is not, try quitting RStudio and relaunching it and do this step again.
    • If it is still not there, talk to your professor. We‚Äôll find it.

11.3.2 Create a new SSH key

Next, we‚Äôll check if your SSH key is already setup:

  1. In the Git/SVN window, look for the SSH key field.
  2. If you SSH Key path is already filled in, then you can skip to the ‚ÄúAdd SSH Keys on Github‚Äù section. If you do not have an SSH key showing, follow the next steps below to make one before moving on.

If you‚Äôve never used SSH keys before, you‚Äôll have to create one, but RStudio makes that fairly painless.

  1. Click on the Create RSA Key button.
  2. You should be able to leave all the defaults as they are. Leave the optional Passphrase field blank.
  3. Click the Create button.
  4. You should see a message that says ‚ÄúKey Created Successfully‚Äù and it might show you a pretty picture. Click OK.

11.4 Add SSH key in Github

Finally, we‚Äôll save part of the contents (the public key) of that key in your Github account. This is so Github ‚Äúknows‚Äù about your computer.

To access your ‚ÄúPublic‚Äù key, do the following:

  1. In the RStudio Git/SVN window, click on the View public key button.
  2. This will open a new window with the contents of your public key.
  3. Copy the full contents of the public key.

  3. Click New SSH key or Add SSH key.

  4. In the ‚ÄúTitle‚Äù field, add a descriptive label for the new key. Name it something like ‚ÄúPersonal MacBook Air‚Äù to indicate its your personal computer.

  5. In the ‚ÄúKey‚Äù field, paste ( Command+V)the public key you copied from RStudio into the box.

  6. Click Add SSH key.

  7. If prompted, confirm your GitHub password.

Let‚Äôs make sure everything is hooked up correctly between our local machine and remote Github account.
